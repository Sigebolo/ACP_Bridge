## 📋 任务描述

根据《心跳回忆 2.0》架构文档第 10 章节，为平台开发支付与订阅管理系统。
由于本项目基于 OpenClaw 运行，支付系统需与 Agent 的权限控制挂钩。

---

## 🎯 核心功能需求

### 1. 订阅档位管理 (Tiers)
- **Free (免费版)**: 限制每日消息数、仅限 1 个伴侣。
- **Heartbeat ($9.9/mo)**: 3 个伴侣、无限消息、每日 10 条语音、3 张照片。
- **True Love ($24.9/mo)**: 10 个伴侣、无限语音/照片、深度记忆解锁。

### 2. 虚拟商店 (Gift Shop)
- 实现虚拟货币（Coins）的充值逻辑。
- 礼物购买触发伴侣独特反应的接口定义。

### 3. 支付集成
- **初期方案**: 建议先实现 Stripe 或 支付二维码上传/手动确认（MVP 阶段）。
- **进阶方案**: 接入 Stripe Webhook 或 支付宝/微信支付接口。

### 4. 数据持久化
- 用户的订阅状态和余额需记录在 `credentials/user_billing.json`。
- Agent 需要能通过 Skill 读取当前用户的权限级别。

---

## 🛠️ 技术要求
- **后端逻辑**: 开发一个 `companion-billing` skill。
- **前端交互**: 在 Web Canvas (HBM-005) 中增加“充值/订阅”入口。
- **API**: 与 Telegram Bot 交互，通过 `/bill` 或 `/topup` 命令查看余额。

---

## ✅ 验收标准
1. 用户可以通过 Web 或 Telegram 查看当前订阅等级。
2. 实现模拟支付流程（输入测试 Key 完成状态更新）。
3. 当用户余额不足或超过订阅限制时，Agent 能给出相应的提示引导。
