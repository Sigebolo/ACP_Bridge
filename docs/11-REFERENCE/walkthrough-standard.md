# Walkthrough Standard

## Walkthrough目的

Walkthrough是任务完成后的标准化收口文档，用于：
- 记录任务执行过程和关键决策
- 为后续Agent提供可复用的经验
- 确保知识积累和传承
- 提供可验证的完成证据

## Walkthrough模板

每个任务完成后必须创建walkthrough文档，文件名格式：`TASK-{ID}-walkthrough.md`

```markdown
---
title: "TASK-{ID} Walkthrough: {任务标题}"
task_id: "TASK-{ID}"
agent: "执行Agent名称"
start_date: "开始日期"
completion_date: "完成日期"
evidence_depth: "L1-L4"
related_ssoT: "Feature SSoT条目链接"
---

## Walkthrough质量标准

### 内容完整性
- [ ] 任务概述清晰完整
- [ ] 执行过程详细记录
- [ ] 关键决策有理有据
- [ ] 技术实现描述准确
- [ ] 测试证据充分

### 结构规范性
- [ ] 使用标准模板
- [ ] 章节结构完整
- [ ] 格式规范统一
- [ ] 链接有效准确

### 实用性要求
- [ ] 可复用经验明确
- [ ] 问题解决方案具体
- [ ] 改进建议可执行
- [ ] 相关文档同步更新

## Walkthrough审核流程

1. **自审核**: 执行Agent完成后自审核
2. **技术审核**: 相关技术Agent审核技术内容
3. **质量审核**: 质量保证Agent审核文档质量
4. **归档**: 审核通过后归档到`docs/10-WALKTHROUGH/`
